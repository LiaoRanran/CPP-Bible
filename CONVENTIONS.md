# 编写规范与约定 · CONVENTIONS

> 本书所有章节强制遵循本规范。任何偏离都需在章末「偏离说明」标注。

## 1. 立场分层标签（必须区分，禁止混淆）

| 标签 | 含义 | 用途 |
|---|---|---|
| `[标准]` | ISO C++ 语言/库规定 | 引用标准条款（如 `[expr]`、`[class.virtual]`） |
| `[实现]` | 编译器/标准库实现细节 | GCC/Clang/MSVC、libstdc++/libc++/MS STL 行为 |
| `[平台]` | OS / ABI / 硬件相关 | Itanium C++ ABI、Windows x64、Linux ELF |
| `[经验]` | 工程共识 / 个人判断 | “通常建议…”“实践中…” |

**规则**：凡是 `[实现]`/`[平台]` 内容，必须标注具体实现（如 `[实现·GCC13]`、`[平台·x86-64 Itanium ABI]`），不得伪装成标准规定。

## 4. 图示约定

- **ASCII 图**：用于内存布局、对象模型、调用栈、依赖图、时序图。使用等宽框线（`┌─┐│└┘├─┤`），节点标明地址/偏移/大小。
- **Mermaid**：流程图 `flowchart`、时序图 `sequenceDiagram`、类图 `classDiagram`、状态图 `stateDiagram-v2`、思维导图 `mindmap`。保存源码于 `Mermaid/`，正文嵌入 ` ```mermaid `。
- **UML 类图**：用 Mermaid `classDiagram` 表达继承/组合/依赖/聚合。
- **汇编图**：用 ` ```asm ` 或 ` ```x86asm `，标注编译器/标志/语法。
- **内存图**：用 ASCII 或 Mermaid `graph` 表达段/堆/栈。

## 5. 代码规范

- 所有代码可编译（C++23，`-std=c++23 -O2 -Wall -Wextra`）。
- 禁止 `using namespace std;` 在头文件；示例正文可局部使用以便阅读，但工业案例必须显式 `std::`。
- 工业案例禁用 Hello World；采用服务器/引擎/DB/网络/渲染/交易等真实场景。
- 示例代码放入 `Examples/<章号>_xxx.cpp`，正文用 ` ```cpp ` 引用关键片段并标注文件路径。
- 错误示例统一前缀 `// ❌`，正确示例 `// ✅`，并说明原因。

## 6. 交叉引用与索引维护

- 章节内引用：`⟶ Book/part03_language/ch20_reference_pointer.md`
- 术语首次出现加粗并链 `Index/GLOSSARY.md#odr`。
- 每章写完：① 在 `INDEX.md §1` 登记；② 在 `INDEX.md §2` 依赖图补边；③ 新术语入 `GLOSSARY.md`；④ 相关知识点在 `CROSSREF.md` 建双向链；⑤ 面试题/误区/FAQ 入对应库。
