## 一、统一章节模板（20 元素）

每节（section，通常 `###`）必须尽量覆盖以下 20 个元素。历史/概览类章节可省略
不适用的元素（如"汇编分析"），但需在章节开头注明省略理由。顺序可按叙述流畅调整，
但建议固定顺序以利扫描。

| # | 元素 | 说明 / 产出物 |
|---|------|--------------|
| ① | 定义 | 一句话精确定义，区分标准定义与个人解释 |
| ② | 历史背景 | 该特性为何出现，解决了什么痛点 |
| ③ | 为什么出现 | 设计动机、替代方案为何不够 |
| ④ | 标准规定 | 引用标准条款（如 `[dcl.fct.def.default]`），注明版本 |
| ⑤ | 编译器实现 | GCC/Clang/MSVC 各自如何实现、扩展、差异 |
| ⑥ | 内存模型 | 对象在内存中的布局/生命周期/所有权 |
| ⑦ | 底层原理 | 编译期/运行期机制、ABI 层面 |
| ⑧ | 汇编分析 | 关键代码 `g++ -O2 -S` 反汇编对照（核心章节必做） |
| ⑨ | STL 联系 | 与标准库组件的关联 |
| ⑩ | 工程应用 | 真实工业场景（服务器/游戏/数据库/OS/交易） |
| ⑪ | 常见错误 | 典型误用、UB、陷阱 |
| ⑫ | 面试问题 | 高频考点 + 参考答案要点 |
| ⑬ | 最佳实践 | 现代 C++ 推荐写法 |
| ⑭ | 性能分析 | 时间/空间复杂度、缓存、分支、实测数据（注明来源/示意） |
| ⑮ | 示例 | **工业级案例，禁止 Hello World**；每节 ≥5，重点节 ≥20 |
| ⑯ | 工业源码引用 | 引用 libstdc++/libc++/MS STL/Boost/Chromium/UE 等片段 |
| ⑰ | 对比其他语言 | Rust/Go/Java/C#/Python 等对比（体现设计权衡） |
| ⑱ | 练习题 | 由浅入深 3–5 题 |
| ⑲ | 总结 | 一图/一表速记 |
| ⑳ | 推荐阅读 | 标准提案编号、书籍章节、CppCon 演讲 |

## 二、四大信息层必须区分（严禁混淆）

每处陈述明确归属一层，用前缀或措辞区分：

1. **[标准]** — ISO C++ 标准规定，所有合规实现必须满足。
2. **[实现]** — 某编译器/标准库的具体做法（GCC/libstdc++ 等），可能变。
3. **[平台]** — OS/ABI/调用约定相关（x86-64 System V、Windows x64 等）。
4. **[经验]** — 委员会/专家个人建议，非强制。

> 例：「[标准] `sizeof(char)==1`。[实现] GCC 在 x86-64 上 `int` 为 4 字节。
> [平台] Windows x64 与 System V 的 `this` 传递寄存器不同。[经验] 优先用
> `std::size` 而非 `sizeof(arr)/sizeof(arr[0])`（Scott Meyers 风格）。」

## 三、编译器 / STL 横向对比策略

凡是涉及"行为/性能/限制"的内容，必须给出三方对比（至少语言核心层）：

| 维度 | 对比对象 |
|------|---------|
| 编译器 | GCC、Clang/LLVM、MSVC |
| 标准库 | libstdc++（GCC）、libc++（Clang/Apple）、MS STL（MSVC） |
| 标准版本 | 标注特性所属最低标准（C++11/14/17/20/23/26） |
| 实验特性 | C++26 未定稿内容标注 `[实验]` 与提案号（Pxxxx） |

**对比表模板**（示例）：

```
| 特性 | GCC | Clang | MSVC | 标准 |
|------|-----|-------|------|------|
| Concepts | 10+ | 10+ | 19.30+ | C++20 |
| 模块 | 11+(TS) / 14+ | 16+ | 19.28+ | C++20 |
```

## 四、ASCII 图约定（用户明确许可）

本书图表以 ASCII 为主（用户许可），关键图优先用 Unicode 框线（`┌─┐│└┘├┤┬┴┼`）
保证对齐。图后必须配文字说明与图例。允许图类型：

- **内存布局图**：对象/容器在内存中的字节排列（偏移 + 字段）。
- **调用栈图**：函数调用链与栈帧。
- **生命周期图**：对象构造→使用→析构时间线。
- **时序图**：对象/线程交互（用箭头 `──►`）。
- **编译流程图**：源码→预处理→编译→汇编→链接。
- **模板实例化流程图**：实参推导→替换→两阶段查找→实例化。
- **同步图（并发章）**：线程 A/B 的 release-acquire 配对、happens-before。

图例示例：
```
       0        8        16  (字节偏移)
      ┌────────┬────────┬────────┐
      │ vptr   │ base_  │ der_   │   <- der_ 在派生类子对象
      └────────┴────────┴────────┘
       ↑虚表指针(8B, 所有平台常见)
```

## 五、代码规范（书中示例代码）

- 全书面向 **C++23** 写作；用到 C++20 前特性时注明最低版本。
- 示例必须可编译（标注编译器与 `-std=`）。禁止未定义行为示例当作"正确示范"。
- 工业级案例：用 `namespace industrial`，避免 `using namespace std;`（教学片段可破例并注明）。
- 命名：类型 PascalCase，函数/变量 camelCase 或 snake_case 统一（与所在案例一致），
  成员变量加 `m_` 或后缀 `_`（按案例惯例）。
- 每个示例标注：①目的 ②关键点 ③期望输出（若重要）。
