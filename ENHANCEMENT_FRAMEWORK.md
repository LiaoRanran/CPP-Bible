# C++ 圣经 · 六维深度增强框架（ENHANCEMENT_FRAMEWORK）

> 目标：把《现代 C++ 终极圣经》从"好教材"（讲清是什么）升级为"世界级专家教材"（讲清为什么——设计动机、标准史、三标准库源码、工业实战、性能证据、知识连接）。
> 本文档定义**六维评分卡 + 附录模板 + 章节优先级 + 覆盖追踪 + CI 红线**，是可复用的工程规范。

---

## 0. 核心纪律（不可逾越）

1. **正文精简，深度沉淀附录**：与全书"正文最简洁、附录承载细节"红线一致。六维深度一律以新增附录（如 `## ⑪ …深度增强`）承载，不膨胀正文。
2. **不引入 CI 红**：
   - 深度片段用 ```` ```cpp ```` 围栏但**不含 `int main`** → 被 `compile_all.py --main-only` 跳过（已核实）。
   - 新增 Mermaid 必须经 `tools/mermaid_lint.py --check` 通过（纯 Python、零依赖、秒级）。
   - 任何断言必须**可移植**：禁止写死具体方言整型（`long long`/`long`）或精确复合布局 `sizeof==`；改用标准类型别名（`std::ptrdiff_t`/`std::size_t`）与可移植下变量（`sizeof>=`/`std::is_empty_v`）。
3. **真实，不伪造**：
   - 性能数字必须来自本地 mingw1530 GCC 15.3.0（`/c/Qt/Tools/mingw1530_64/bin/g++.EXE`）实跑，标注编译器/旗标/环境/样本量。
   - 汇编实证须真实 `objdump -d -M intel -C`，绝不编造符号。
4. **跨章节一致性**：同一概念（如 noexcept 移动）在多章出现时，指定**一章为"总论"**（目前是 ch115），其余章为"语境补篇"并互相交叉引用，避免事实分叉。

## 5. 工作流（新增一章六维深度）

1. **定位**：确认该概念是否有"总论"章；若无，本附录即总论；若有（如移动语义=ch115），本附录须是"<语境>补篇"并交叉引用。
2. **实跑基准**（D5）：用 mingw1530 编译 `-O2 -std=c++17` 实跑，取真实数字（参考 ch32 的 `_bench_move.cpp` 方法，跑完即删临时文件）。
3. **写附录**：套用 §2 模板，D3/D6 用 Mermaid。
4. **本地门禁**：`python tools/mermaid_lint.py --check` 必过；`grep -c "int main"` 确认未新增可编译块。
5. **交叉引用**：在总论章与该补篇章互加链接，防事实分叉。
6. **登记**：更新 §4 追踪表 + MEMORY.md。
7. **提交**：本地 `git commit`（gpgsign=false）；推送前确认 CI #196 等历史运行已绿，避免叠加卡死。

## 6. 真实基准方法学（D5 可复用）

- 编译器：`C:/Qt/Tools/mingw1530_64/bin/g++.EXE`（`__VERSION__` 打印 15.3.0）。
- 旗标：`-O2 -std=c++17`（贴近发布）。
- 对照设计：构造 `noexcept` 移动类型 vs **潜在抛出**移动类型（用户声明移动构造不标 `noexcept` → 触发 vector 复制回退），同环境跑同负载。
- 报告四项：编译器、样本量、各对照耗时(ms)、倍数；并解释差距来源（内存搬运量/alloc/cache）。
- ch32 实测（mingw1530 -O2，push_back×20000）：1/4/16/32 KiB 元素下，复制回退稳定慢 **2.75x–3.50x**，绝对浪费随元素尺寸线性放大。
