## 二、打磨方向（竣工后，有信息增量的工作）

以下每一项独立可做，按优先级排序。不违红线：每条工作都有**具体的信息增量来源**，不做关键词堆砌/附录批量追加/空洞 cpp 块。

### 优先级 1：代码块自包含化（工程可维护性）

**现状**：51 个断言 WARN 全因代码块依赖前文类型（`void_t`/`Empty`/`Node`/`Big` 等），读者无法"抄一段就跑"。密度审计也受此拖累（块间依赖降低独立性得分）。

**做法**（每章 5-15 分钟）：
- 在依赖前文类型的块内补 `// 前置定义见上文` 注释 + 自包含版本的类型别名
- 对高频依赖（如 `void_t`）在 chapter_compile_check PRELUDE 里统一注入
- 范围：聚焦 51 个 WARN 涉及的 ~20 章

**信息增量**：可抄可编译 → 工程可用性从\"参考\"升到\"开箱即用\"。

### 优先级 2：真编译期常量 + 汇编验证（底层事实）

**现状**：部分性能断言用 `~5ns`/`~50cycles` 风格，无实测平台、无编译器版本、无汇编对应行。

**做法**：
- 针对关键 idiom（虚函数开销、CRTP vs 虚函数、`std::function` vs 函数指针、EBO 实际生效、`std::vector` 小对象优化等），用本机 GCC 13.1 `-O2` 实测 + Compiler Explorer 链接佐证
- 每条包含：源 → 汇编关键行（对应行号）→ 周期数/纳秒（注明 HCLK 与测量方法）
- 不编造数字；无法实测的注明"平台相关，见 Compiler Explorer: [链接]"

**信息增量**：从"大约"到"实测可复现"，契合项目早期要求的纳秒级时序 + 寄存器位值风格。

### 优先级 3：真实工业源码引用（可查证性）

**现状**：已有 Chromium/Abseil/LevelDB/ClickHouse 章，但部分引用是类名/函数名罗列，缺具体文件和行号。

**做法**：
- 对已有工业章（ch130_chromium_abseil、ch132_leveldb_rocksdb、ch133_clickhouse_redis 等）补 GitHub permalink（具体文件 + 行号 + commit hash）
- 对设计模式章（Part 12）补一个真实开源项目中的实际用例链接

**信息增量**：从"据说用了"到"这一行就是"。

### 优先级 4：弱章精准补强（非灌水）

**现状**：密度最低两章 ch52_ebo（20/30）和 ch135_patterns_intro（20/30），但都在"完整"线以上。需判断是天然窄话题还是真有知识缺口。

**ch52_ebo 可补方向**：
- 本机 GCC 13.1 实测 `[[no_unique_address]]` vs 空基类 EBO 在不同 `-std=` 下的行为差异表（C++17/20/23）
- `std::compressed_pair` / `std::unique_ptr<T, Deleter>` 的 EBO 实际运用 + libstdc++ 源码引用

**ch135_patterns_intro 可补方向**：
- 设计模式分类学的历史演变（GoF 1994 → C++ 惯用法 → C++20 Concepts 替代方案）
- 用一个贯穿实例（如日志系统）展示从"不用模式"→"用错模式"→"正确模式"→"现代 C++ 替代"的演化路径

**做法**：先跑 `suggest.py` 对这两章出建议报告，人工判断哪些是真缺口再动刀。

### 优先级 5：C++26 追踪（持续更新）

**现状**：ch09_cpp26 已有基础内容，但 C++26 仍在演进（contracts、reflection、pattern matching 等关键特性状态持续变化）。

**做法**：
- 每季度检查一次 WG21 最新进展（open-std.org 邮件列表 + cppreference C++26 页面）
- 仅更新**已投票进入工作草案**的特性（不编造提案状态）
- 格式：特性名 + 提案号（可查证）+ 简短示例 + 编译器支持矩阵

**信息增量**：持续更新一个移动靶，时效性本身就是信息增量。

### 优先级 6：学习路径互动化（已有基础）

**现状**：`learning_paths.md`（嵌入式/后端/考研 3 条路径）+ `docs/learning_path_viz.html`（DAG 可视化）已就位，但路径是静态的。

**做法**：
- 在静态站点中嵌入 `learning_path_viz.html`（当前已在 docs/，挪到 site_out/ 并加导航入口）
- 可选：按路径生成"精选阅读顺序"（从 147 章中选 30-50 章），标注每章"为什么重要"和预计阅读时间

**信息增量**：从"参考书"到"有指南的参考书"。
